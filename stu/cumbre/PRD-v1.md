# Cumbre — 基金智投跟投平台 · 产品需求文档 v1

---

## 1. 产品定位

**一句话**：全覆盖中国大陆公募基金市场，以信号驱动为核心，自动研判买卖时机，提供自动跟投（模型验证）与手动跟投（用户决策）双模并行的智能投研平台。

**核心理念**：系统负责决策，用户选择信任程度。

---

## 2. 数据来源

| 数据 | 来源 |
|---|---|
| 基金列表、净值、持仓 | 天天基金（东方财富）公开接口 |
| 指数 PE/PB 百分位 | 东方财富 / 中证指数官网 / 乌龟量化 |
| 市场行情 | 新浪财经 / 东方财富 API |
| 覆盖范围 | **中国大陆公募基金全市场**（股票型、混合型、指数型、ETF联接、债券型） |

## 3. 信号系统（核心引擎）

系统的灵魂。多因子打分模型，每日收盘后对**全市场基金**进行评分，产出 Buy / Sell / Hold 信号。

### 3.1 信号因子模型

| 因子 | 权重 | 说明 |
|---|---|---|
| **估值因子** | 40% | PE/PB 历史百分位。百分位越低，买入信号越强 |
| **趋势因子** | 25% | MA20/MA60 均线系统、价格偏离度（超跌/超涨） |
| **动量因子** | 15% | RSI、MACD，避免逆势操作 |
| **质量因子** | 10% | 基金规模稳定性、基金经理任职年限、最大回撤 |
| **情绪因子** | 10% | 市场换手率、发行冰点指标（人少时买） |

### 3.2 信号等级

| 等级 | 综合分 | 含义 | 操作 |
|---|---|---|---|
| S — 强买 | ≥ 80 | 历史低估 + 技术共振 | 优先买入，加仓 30～50% |
| A — 买入 | 65～79 | 估值偏低或合理 | 正常买入，加仓 15～25% |
| B — 持有 | 40～64 | 估值合理 | 持有不动 |
| C — 减仓 | 25～39 | 估值偏高或趋势走坏 | 减仓 20～30% |
| D — 强卖 | < 25 | 高估 + 恶化 | 减仓 50% 以上 |

### 3.3 信号输出

每天净值更新后（约 20:00），系统运行全市场评分 → 生成**信号池** → 按规则推送到两个独立的执行模块。

### 4.1 双模块共享的执行清单

信号池经过策略筛选后，输出为一份统一的**执行清单**，按三种时间维度组织：

| 计划 | 频率 | 定位 |
|---|---|---|
| **日投计划** | 每个交易日 | 捕捉大跌加仓、超卖机会、紧急卖出 |
| **周投计划** | 每周一次 | 主力定投节奏，攒份额 |
| **月投计划** | 每月一次 | 仓位再平衡，长线建仓 |

**这份清单是自动和手动两个模块的共同输入。** 两份推荐内容完全一致，区别在后续处理方式。

清单格式：

```
┌──────────────┬───────┬──────┬──────────┬──────────┬──────────┐
│ 基金名称     │ 动作  │ 强度 │ 建议金额 │ PE分位   │ 说明     │
├──────────────┼───────┼──────┼──────────┼──────────┼──────────┤
│ 招商中证白酒 │ 买入 │ S ★  │ 500-1000 │ 12% 低估 │ 低估推荐  │
│ 易方达蓝筹   │ 买入 │ A ★  │ 200-500  │ 28% 偏低 │ 可分批买  │
│ 天弘沪深300  │ 持有 │ B    │ —        │ 48% 合理 │ 持有不动  │
│ 中欧医疗健康 │ 卖出 │ C ★  │ 减30%    │ 72% 偏高 │ 建议减仓  │
└──────────────┴───────┴──────┴──────────┴──────────┴──────────┘
```

### 4.2 模块一：自动跟投（模型验证工具）⚙️

**定位**：独立的自动化模拟器，目的是**验证信号模型的有效性**。

系统直接读取执行清单，按内置分配规则自动执行，用户不干预。

#### 规则

| 项目 | 说明 |
|---|---|
| **用户配置** | 仅设置**总投入金额** |
| **执行方式** | 按日投/周投/月投的节奏，周期性自动执行 |
| **基金分配** | 按执行清单的信号等级分配资金，**不可修改** |
| **操作** | S/A 级自动买入、C/D 级自动卖出 |
| **干预** | 不可编辑清单，不可干预单笔交易 |

#### 输出报告（周报 / 月报 / 季报）

| 指标 | 说明 |
|---|---|
| 区间收益率 | 本周/月/季的总收益率 |
| 信号准确率 | S/A 信号买入后 N 天上涨比例 |
| 各因子贡献度 | 各因子对收益的贡献分解 |
| 胜率 vs 基准 | 跑赢沪深300的比例 |
| 最大回撤 | 期间最大回撤 |
| 调仓记录 | 每次自动操作的明细 |

### 4.3 模块二：手动跟投（用户决策工具）🔑

**定位**：同一份执行清单，但用户可以编辑后再决定是否执行。

#### 操作流程

1. 查看执行清单（日/周/月 Tab）
2. **编辑推荐列表**：
   - **增**：添加清单外的基金
   - **删**：跳过不想跟的基金
   - **改**：调整买入金额或比例
3. 点击「提交执行」→ 按编辑后的清单在**手动模拟组合**中执行交易
4. 执行结果回显：成交明细

### 4.4 双模块对比

| 维度 | 自动跟投 | 手动跟投 |
|---|---|---|
| 推荐清单来源 | 同一份执行清单 | 同一份执行清单 |
| 编辑能力 | **不可编辑** | **可自由编辑**（增删改） |
| 执行方式 | 周期性自动执行 | 用户确认后提交 |
| 用户配置 | 只设总金额 | 逐项决定 |
| 干预能力 | 不可干预 | 完全控制 |
| 输出 | 收益分析报告（改进模型） | 个人投资收益 |
| 组合 | 独立自动组合（只读） | 独立手动组合 |

## 5. 功能模块总览

### 5.1 市场总览（仪表盘）

- 大盘指数卡片（上证/深证/创业板/科创50）
- **市场温度计**：宽基指数 PE 百分位矩阵（绿/黄/红）
- **全市场信号分布**：今日 S/A/B/C/D 信号各多少只
- **自动组合概览**：收益率、信号准确率快照
- **执行清单待办**：今日待处理的推荐数量

### 5.2 基金筛选与分析

- **全市场基金列表** + 多维筛选（类型/规模/年化收益/夏普/最大回撤）
- 基金详情页：
  - 净值走势图（1月/3月/1年/3年/全部）
  - 业绩指标（各区间收益率、最大回撤、夏普、波动率）
  - **估值曲线**（PE/PB 百分位走势，当前点位标记）
  - 持仓明细 + 行业分布
  - 基金经理信息
  - **当前信号**（S/A/B/C/D + 评分）

### 5.3 自动跟投（系统运行）⚙️

- **总览面板**：投入总额、当前市值、总盈亏、收益率、运行天数
- **持仓列表**（系统自动分配的基金及比例、盈亏）
- **调仓记录**（每次自动操作的流水）
- **收益报告**：周报 / 月报 / 季报
  - 区间收益率
  - 信号准确率
  - 因子贡献分解
  - 与基准对比
- **配置**：设置总投入金额
- ⚠️ 不可修改分配比例，不可干预单笔交易

### 5.4 手动跟投 — 执行清单（核心入口）🔑

- 三个 Tab：**日投计划 / 周投计划 / 月投计划**
- 每个计划展示系统推荐列表
- **可编辑**：增加基金、删除基金、修改金额/比例
- 「提交执行」按钮 → 推入手动模拟组合
- 提交后回显：成交明细

### 5.5 手动模拟组合

- 总览（总投入、市值、盈亏、收益率）
- 持仓列表
- 收益率曲线 vs 基准
- 交易记录

### 5.6 自选管理

- 关注基金 + 自定义分组
- 自选基金的信号变化提醒

## 7. 数据模型

```
Fund              — 基金（code, name, type, company, fund_size, risk_level, manager_id, established_date）
FundNav           — 净值（fund_code, date, nav, acc_nav）
FundValuation     — 估值（fund_code, date, pe, pb, pe_percentile, pb_percentile）
FundHolding       — 持仓（fund_code, stock_code, stock_name, ratio, date）
FundManager       — 经理（id, name, description, tenure, fund_codes）
IndexQuota        — 指数估值（index_code, name, pe, pb, pe_percentile, pb_percentile, date）

Signal            — 信号（id, fund_code, date, score, level, action,
                     factors_detail JSON, created_at）

--- 自动跟投 ---
AutoConfig        — 自动配置（user_id, total_amount, status, created_at）
AutoPortfolio     — 自动组合（user_id, cash, total_invested, market_value, nav_history JSON）
AutoPosition      — 自动持仓（portfolio_id, fund_code, shares, cost_nav, allocation_ratio）
AutoTrade         — 自动交易（portfolio_id, fund_code, type, amount, shares, nav, date, plan_type, signal_id）
AutoReport        — 收益报告（id, user_id, period_type(weekly/monthly/quarterly),
                     period_start, period_end, stats JSON, created_at）

--- 手动跟投 ---
ExecutionItem     — 执行清单项（id, user_id, plan_type(daily/weekly/monthly),
                     fund_code, action, suggested_amount_min, suggested_amount_max,
                     signal_id, status(pending/executed/skipped), date）

ManualPortfolio   — 手动组合（user_id, name, cash, total_invested, created_at）
ManualPosition    — 手动持仓（portfolio_id, fund_code, shares, cost_nav）
ManualTrade       — 手动交易（portfolio_id, fund_code, type, amount, shares, nav, date, execution_id）

--- 通用 ---
Watchlist         — 自选（user_id, fund_code, group_name, created_at）
```

## 9. 技术方案

| 层 | 方案 |
|---|---|
| 后端 | Node.js + Express + TypeScript |
| 前端 | React + TailwindCSS + Recharts |
| 数据库 | PostgreSQL（数据量大，推荐）或 SQLite（开发用） |
| 数据采集 | node-cron 定时任务，每日盘后拉取 |
| 信号引擎 | 纯后端计算，每日收盘后运行 |
| 自动执行 | node-cron 定时触发，按日/周/月执行策略 |
| 图表 | Recharts + 简易表格 |
