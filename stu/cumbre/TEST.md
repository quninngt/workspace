## 4. 端到端工作流测试

### 4.1 自动跟投完整流程

```
注册用户 → 设置日投金额(¥100) → 等待信号生成 → 自动执行
→ 验证持仓创建 → 验证交易记录 → 验证组合市值
```

**结果**: 信号引擎生成 → auto_executor 处理 → 10 只基金各 ¥10 → 组合市值 ¥100 ✅

### 4.2 手动跟投完整流程

```
从信号生成 → 查看执行清单 → 确认提交 → 查看手动组合
→ 验证持仓 → 验证交易记录 → 验证状态变更
```

**结果**: 192 条生成 → 7 条 buy 执行 → 7 条 hold 跳过 → 组合 ¥7,000 ✅

### 4.3 数据采集 + 信号生成流程

```
触发基金同步 → 触发 NAV 同步 → 触发信号生成 → 验证信号分布
```

**结果**: 信号从 184 增至 376（二次运行），A 级从 10 增至 20 ✅

## 6. 已知问题

| # | 问题 | 影响 | 优先级 |
|---|------|------|--------|
| 1 | **信号集中在 B 等级**：估值因子使用沪深 300 指数替代，94.6% 评分 B | 买卖信号稀疏 | P1 |
| 2 | **NAV 覆盖率低**：仅 192/26,710 有净值数据 | 信号覆盖不足 | P1 |
| 3 | **持仓数据稀疏**：仅 5 只基金有持仓 | 基金详情页数据少 | P1 |
| 4 | **基金详情字段缺失**：company/risk_level/manager 多为 NULL | 信息不完整 | P1 |
| 5 | **回测收益率 0%**：因全 B 信号无交易触发 | 无法评估策略 | P1 |
| 6 | **无密码找回机制** | 用户体验 | P3 |
| 7 | **无 Token 自动刷新** | 用户体验 | P3 |

## 7. 测试结论

**整体评价**: 系统核心功能完备，API 通过率 96.6%。所有 24 个 API 端点正常工作，10 个前端页面路由完整，前后端联调通过。

**主要风险**: 信号分布偏窄（P1）影响自动/手动执行的有效性，建议优先解决基金面 PE/PB 数据源问题。

**建议优先修复**:
1. 扩展 NAV 数据覆盖（从 192 → 500+ 只基金）
2. 校准信号引擎因子参数，使评分分布更分散
3. 补充基金详情字段数据源
